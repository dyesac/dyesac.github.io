なぜだっ！！pipが動かない \_(꒪ཀ꒪」∠)_ proxyだってちゃんと設定したのに…、の時に。

Posted by dyesac in memo   

環境

MacOs 10.14.2
pyenv 1.2.0
Python 3.6.0
pip 18.1

Proxyはちゃんと通っているよね？

proxyの設定方法

".bash_profile"にProxyの情報を記載します！

$ vim  ~/.bash_profile

# .bash_profile
HTTP_PROXY = http://xxx.xxx.xxx.xxx:xxxx
HTTPS_PROXY = $HTTP_PROXY
ALL_PROXY = $HTTP_PROXY

で、".bash_profile"を読み込み直します

$ source ~/.bash_profile
windowsの場合

コマンドプロンプトから環境変数を設定します。

$ set HTTP_PROXY=http://{UserName}:{Password}@{proxy.server}:{port}
$ set HTTPS_PROXY=http://{UserName}:{Password}@{proxy.server}:{port}
pipのオプションでする場合
$ pip install <パッケージ名> --proxy http://xxx.xxx.xxx.xxx:xxxx

pipの不能は突然に

使えていたはずのpipが使えない？？ネットワーク環境の変更に伴う、proxyの設定だったり、一部ネットワーク環境ではcntlmを利用していたり、pyenvの環境によって違う？など、、適当に使ってきた影響でぐちゃぐちゃで何が原因かさっぱり理解できない状況になっていました。

症状について

$ pip install <package>
Collecting <package>
  Could not find a version that satisfies the requirement <package> (from versions: )
No matching distribution found for <package>

と、、versionがない？と。。そんなはずはない！！！

過去にはネットワークのエラーが原因で、pipが利用できないことがあったので、同様に、ネットワークが原因でこんなエラーメッセージが出てるのかも？？とおもったのですが、

$ pip search <package>

は通るのです。。ちゃんと。

原因がわからないまま月日は流れ、どうしても必要なパッケージがある時は、pypiからDLしてきて手動でインストールするという、苦行をしておりました。関連するパッケージが複数あることもしょっちゅうで、こんなに関連パッケージあるんだーーーっ（いつになったら目的のパッケージが入るのじゃorz）と勉強になったものの、やってられない日々が続きまして。

が、やっと解決できたので、同じような症状に嘆き悲しむ方々の少しでも助けになればと思い、対処した内容について記録しておきます

原因はSSL/TLSでの通信エラー

詳細な原因については理解しきれていないのですが、SSL/TLSで利用している証明書の有無が原因だったようです。 homebrew でインストールした open-ssl の証明書の配置されるディレクトリに問題がある？のか。

pip(python) からSSL/TLS通信する際の証明書が見つからず、エラーが発生していたようです。

英語でのやりとりが難しく、ちゃんと理解できていないかもしれないので詳しくはこちらのやりとりを確認してもらいたいですが、確信の部分をすっごく意訳すると。

（・・・略・・・）

（回答者）以下、証明書、鍵はがあるか？

"/etc/ssl/certs/ca-certificates.crt",
"/etc/pki/tls/certs/ca-bundle.crt",
"/etc/ssl/ca-bundle.pem",
"/etc/ssl/cert.pem",
"/usr/local/share/certs/ca-root-nss.crt",
"/usr/local/etc/openssl/cert.pem",

（質問者）以下のものしかない

"/etc/pki/tls/certs/ca-bundle.crt",
"/usr/local/etc/openssl/cert.pem",

（回答者） きっと"/etc"にすれば直るよー

（・・・略・・・）

で、解決したそうです。

実際、何をしたか

"/usr/local/etc/openssl/cert.pem"

を、以下にもコピーしました。

"/etc/ssl/cert.pem"

で、再起動したらpipが正常に動くようになっていました ∩( ･ω･)∩ばんざーい
