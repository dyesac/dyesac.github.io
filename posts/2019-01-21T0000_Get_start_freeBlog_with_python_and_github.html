Posted by dyesac in memo   

かなしい思いでは置いておいて、、、

Pelican のインストール

pipでインストールします。markdownで書く場合は、一緒にmarkdownも。

~ $ pip install pelican markdown

pelican用のディレクトリを作成します

~ $ mkdir Peclican
~ $ cd Pelican

pelicanには、ブログの雛形を生成してくれる便利なコマンドがあるので pelican-quickstart で質問に答えていくとブログが出来上がります。

Pelican $ pelican-quickstart

Pelicanのテーマを導入

テーマファイルのDL

同フォルダに ./themes を作成してその中に、テーマファイルを格納します。テーマは コチラ でたくさん公開されています。いくつか中でシンプルで多機能そうな voidy-bootstrap をまずは採用してみることにしました。

Pelican $ mkdir themes
Pelican $ cd themes
themes $ git clone https://github.com/robulouski/voidy-bootstrap.git

テーマの反映

テーマを反映するためにはpelicanconf.py にテーマの設定を追記してあげます

themes $ cd ../
Pelican $ vim pelicanconf.py

で、適当な位置に以下を一行追加して保存します。

THEME = './themes/voidy-bootstrap/'

これでテーマが反映されます。

記事ページの生成と確認

記事は ./content 以下にテキストファイルとして reStructureText(.rst) またはMarkdwon(.md)で作成していきます。

Pelcian $ vim content/first_post.md

で、タイトルやタグなどのメタデータはお作法があります。テキストファイルのあたまに{metadata}: の形式で記述してあげます。

Title: タイトル
Date: 2019-01-01 00:00
Tags: Pelican, Python, blog
Category: memo
Slug: first_post
Authors: dyesac
Summary: ポストのサマリ

Githubのリポジトリをつくる

経緯（かなしい過去の思い出）

その昔、、RED HAT のOPENSHIFTというサービスにWordpressをいれてしばらくブログとして活用していました。PVはすくなく、自分のメモがわりにつけていたブログには無料枠の範囲でも十分な性能で、インターネッツの知識に乏しい私でもなんとなーく、運用ができるくらいで、非常に使い勝手の良く使わせてもらっていたのですが、、

突然のバージョンアップと無料枠の変更 orz

実際は、半年ほど前から「提供内容かわるからバックアップとるなり、なんなりしなよー」というメールがきていたのですが、、なにせ英語だからちゃんと読まずに読み飛ばしていました。結果、キレイに全データは消えてなくってしまいましたとさ orz

多くはありませんが3年ほどかけて書きためた（北海道一周旅行の軌跡をGoogleMapにプロットしたヤツどっかに残ってないかな〜。。）テキストが消えてしまった喪失感から、しばらく、、メモや記録を残すことから遠ざかってきました。。

Github のFREEプランでプライベートリポジトリが利用可能に！

なんて、2019年、新年のニュースにワーワー、よく分からないけど、すごーい、使ってみよう！となったのがキッカケで Github Page ってんのが、静的サイトがホスティングできるし、ソースはそのまま、プライベートリポジトリに置いておけば、バックアップも完璧じゃーーーん！！サイコー！うひょー！ということで、気をとりなおして ブログを再開してみることにいたしました。本当に、データの管理は大切ですね。
